## Design Decisions

| Decision | Rationale |
|----------|-----------|
| Symbol extraction not a rejection criterion | extract_symbol → slug → market_id always produces a value; no market should be dropped for this |
| First-failing-rule wins | Predictable, deterministic, easy to test rule priority |
| All RejectionReason keys always in breakdown | Caller never needs to check key existence; zero-count is explicit |
| Stateless/no persistence | Clean, testable, composable with any input source |
| Reuses duration constants from market_fetcher | Single source of truth for [240, 360]s window |
