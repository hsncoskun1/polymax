## Problem Solved

Two separate candidate selection paths existed:

| Path | Location | Called by |
|------|----------|-----------|
| `_is_5m_candidate()` raw-dict gate | `market_fetcher.py` | `fetch_candidates()` only |
| `DiscoveryService.evaluate()` | `market_discovery.py` | `MarketSyncService`, `/discover` endpoint |

`fetch_candidates()` was **dead code** — no production caller used it.
Both paths implemented overlapping but separately maintained selection logic.

## Decision

**Removed** `fetch_candidates()` and `_is_5m_candidate()` from `PolymarketFetchService`.

Rationale:
- No production caller (sync, discover endpoint) used `fetch_candidates()` — it was already bypassed in v0.5.2
- `DiscoveryService` is the declared single authority for candidate selection
- Keeping dead code with duplicate logic creates maintenance risk and reader confusion
- Removal is safe: no API surface, no downstream dependency
