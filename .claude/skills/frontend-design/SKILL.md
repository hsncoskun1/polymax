# Frontend Design

Create clean, responsive, and accessible frontend interfaces.

## Principles

- Mobile-first responsive design
- Semantic HTML structure
- Clean CSS with consistent spacing and typography
- Accessible by default (proper labels, contrast, keyboard nav)
- Minimal JavaScript - use it for behavior, not layout

## Process

1. Understand the UI requirements and target devices
2. Structure with semantic HTML first
3. Style with CSS (prefer CSS Grid/Flexbox for layout)
4. Add interactivity with vanilla JS or the project's framework
5. Verify with preview tool at different viewport sizes

## Guidelines

- Use rem/em for sizing, not px (except borders)
- Design for 320px minimum viewport width
- Ensure 4.5:1 contrast ratio for text
- Use system font stack unless a specific font is required
- Keep layouts simple: prefer single-column on mobile
- Test with preview_screenshot and preview_inspect for verification
- Use preview_resize to check responsive behavior
