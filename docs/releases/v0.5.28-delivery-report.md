## Authoritative source map (this surface)

| Information | Source |
|-------------|--------|
| Backend version string | `config/default.toml [app] version` |
| Backend version in API | `GET /health` → `version` field (now reads from config) |
| Backend version in UI | `AdminPanel` fetches `/health` on mount → "Backend" chip |
| Frontend version | **No authoritative source** — `package.json version = "0.0.0"` (Vite default, meaningless); no build-time injection; chip stays `"—"` |

## Why this approach

- `AdminPanel` makes a single `/health` fetch on mount — self-contained, no state lifting required
- `HealthBadge` continues to poll every 30 s for liveness check (separate concern, not duplicated)
- `fetchHealth()` added to `api.ts` keeps all backend calls in one place
- Frontend version chip stays `"—"` — `package.json version = "0.0.0"` is the Vite default and not a meaningful version; fabricating a value would be worse than an honest placeholder
