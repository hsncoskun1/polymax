## localhost alias rationale

`_build_cors_origins()` always appends `http://localhost:{port}` when the config
host is not already `localhost`. Browsers treat `127.0.0.1` and `localhost` as
distinct origins in some security contexts (cookies, service workers, credentialled
requests). Keeping both prevents silent auth failures when a user opens
`http://localhost:5173` instead of `http://127.0.0.1:5173`.
