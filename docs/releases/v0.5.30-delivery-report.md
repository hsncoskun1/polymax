## Startup path analysis

### Files inspected
- `launcher/main.py`
- `README.md`
- `config/default.toml`
- `frontend/vite.config.ts`
- `backend/app/core/config.py`

### Startup path map

| Path | Config-driven | Readiness poll | Opens browser | When to use |
|------|--------------|----------------|---------------|-------------|
| `python launcher/main.py` | ✅ Backend host/port | ✅ Polls `/health` and `/` | ✅ | **Always — recommended path** |
| Manual backend `python -m uvicorn ...` | ❌ Hardcoded CLI args | ❌ | ❌ | Isolated backend dev/debug |
| Manual frontend `npm run dev` | ❌ `vite.config.ts` | ❌ | ❌ | Isolated frontend dev/debug |

### Authoritative startup path: launcher

- Launcher reads `config/default.toml [backend]` host/port for uvicorn args
- Launcher reads `config/default.toml [frontend]` host/port for readiness polling and browser URL
- Launcher does NOT pass host/port to Vite — Vite uses `vite.config.ts` hardcoded values
- Launcher polls `http://{FRONTEND_HOST}:{FRONTEND_PORT}/` from config → must match vite.config.ts

### Behavioural difference between launcher and manual startup

| Behaviour | Launcher | Manual |
|-----------|----------|--------|
| Backend host/port source | `config/default.toml` | Hardcoded CLI args |
| Frontend port source | `vite.config.ts` (Vite, as always) | `vite.config.ts` (same) |
| Readiness detection | ✅ Polls before proceeding | ❌ None |
| Browser auto-open | ✅ Opens `/user` | ❌ Manual |
| Process lifecycle management | ✅ Watches both, exits on crash | ❌ Separate terminals |
| Config change response | Backend follows; frontend does not | Neither follows |
